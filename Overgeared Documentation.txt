FOR 1.21.1 ITS “result”: { “id”: “item_id” }, same with “result_failed”
"blueprint" (OPTIONAL) - required blueprint for the forging item
"requires_blueprint" (OPTIONAL) - if the blueprint is a hard requirement for forging the item
"tier" (OPTIONAL) - which tier of anvil is required to make certain items. Default is iron.
        Following tiers: stone, iron, above_a, above_b
"hammering" - the required hits to forge an item
"has_quality" - whether if the item has a quality or not
"minimum_quality" (OPTIONAL) - the lowest possible quality you can get from a forging minigame, as an example if it's set to "well" its lowest quality will be well forged. 
Following forging qualities: poor, well, expert, perfect, master
"needs_minigame" (OPTIONAL) - if "has_quality" is turned off, it can be set if for items that has a minigame but no quality
"has_polishing" (OPTIONAL) - whether the item needs to be polished on a grindstone or not, default is true
“need_quenching” (OPTIONAL) - whether the item needs to be quenched after crafting or not, default is false for armor items, true for the others
“requires_heated” (OPTIONAL) - if this item needs to be heated for the recipe to work
“transfer_nbt” (OPTIONAL) - if this item’s nbt tags will be transferred to the result item (including damage)
"result_failed" (OPTIONAL) - if has_quality is turned off and needs_minigame is turned on, it can be the resulting item if the player does poorly in the minigame. Does not work if has_quality is true
"quality_difficulty" (OPTIONAL) - sets the minigame’s difficulty, overriding the difficulty normally determined by the blueprint’s quality. Higher values make the minigame harder. If not specified, the blueprint’s quality is used by default.

	Effect

	Flat addition
	mult_base
	Multiplies base value
	mult_total
	Multiplies final value
	

Item Tags
overgeared:knappables
If an item can be knapped
overgeared:heated_metals
Constantly hurt the player when held, needing tongs to handle. Will cool down into another item that was defined in overgeared:cooling recipes.
overgeared:hot_items
Same effect as heated_metals but with different tooltips. Cannot be cooled down into another item.
overgeared:smithing_hammers
For modpack developers to add their own tiered hammers
overgeared:tongs
For modpack developers to add custom items with the tongs behavior
overgeared:tool_parts
For categorizing tool parts
Block Tags
overgeared:needs_copper_tool
overgeared:needs_steel_tool
overgeared:smithing_anvil
For modpack developers to add their own tiered anvils
overgeared:grindstones
What block can be used for grinding items
stone_anvil_bases
What block can be turned into Stone Smithing Anvil
iron_anvil_bases
What block can be turned into Iron Smithing Anvil
tier_a_anvil_bases
What block can be turned into Tier A Smithing Anvil
tier_b_anvil_bases
What block can be turned into Tier B Smithing Anvil
Anvil Tiers
stone
Primitive anvil, accepts copper.
iron
Main anvil used in Overgeared, accepts iron and steel.
above_a
For modpack developers, it accepts anything higher than steel.
above_b
For modpack developers, it accepts anything higher than above_a.


Blueprint Settings
To add a custom blueprint type: add it to availableToolTypes in common config, then define its display name in your lang file.
For example: custom tool type is ARMOR, so you need to put "tooltype.overgeared.armor": "Armor" in the lang file.
It was done like this to allow people to create translations for the tool types without hardcoding the name like past versions.
